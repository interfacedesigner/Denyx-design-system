## 컴포넌트 계층 — Primitives → Composite → Shell

단일 축(Design Theory) 계층. **조합 깊이**가 유일한 분류 기준.

```
Primitives ──→ Composite ──→ Shell
(자립 렌더)    (1단계 조합)    (최종 조립)
```

### Primitives — 자립 렌더 (유일하게 독립 생성 가능)

다른 DS 컴포넌트를 import 하지 않고 **혼자서 렌더**되는 최소 단위. HTML 원소 + 토큰만으로 구성. **3계층 중 유일하게 독립 생성 가능.**

- **판별:** "다른 DS 컴포넌트 import가 있는가?" → 없으면 Primitives.
- **General:** Button · Checkbox · Chip · DataTable · Modal · Select · Tabs · TextField · Toast · Tooltip · Switch · ThemeToggle · LiveTimerCompact · MiniLineChart
- **Parts (Chrome):** SidebarMenuItem · SidebarLogoHeader · DataTableRow · TabButton · HeaderPillButton · WeekHourMatrix · Stage · SubHeaderBar …
- **Parts (AI):** AiCard · AiCaption · AiSectionHeading · AiToneBadge · AiBulletList · AiSendButton · AiAssistantButton · AiToneRow · AiOptionButton · AiSeverityRow …

### Composite — 1단계 조합 (독립 생성 불가)

Primitives를 조합하여 **영역 수준의 UI 단위**를 구성. 독립 생성 불가 — 반드시 Primitives에 의존. **Composite 끼리 import 금지.**

- **판별:** "Primitives만 조합했는가?" → 맞으면 Composite.
- **Chrome:** PageHeader · PageHeaderAiInline · FilterBar · FilterChip · FilterDropdown · Sidebar
- **AI:** AiInlinePrompt · AiChatExchange · AiPromptInput · AiReasoning · AiProposalCard · ProposalSection

### Shell — 최종 조립 (독립 생성 불가)

Composite + Primitives를 **최종 조립**하는 앱 진입점. 독립 생성 불가 — 반드시 하위 계층에 의존. Composite가 다른 Composite를 import하면 Shell로 승격.

- **DashboardLayout** · **DenyxAiWidget** (2개)

### 의존 방향 (역방향 import 금지)

```
Tokens  ←  Primitives  ←  Composite  ←  Shell
```

Tokens는 leaf(누구도 import 안 함). 상위 계층이 하위만 import. 자세한 규칙: [patterns.md](/design-system/patterns.md).

## 카탈로그 개요

전체 카탈로그는 **좌측 트리**가 정본이며, 각 항목의 Docs에 prop·예시·When to use/NOT이 있습니다. 구성:

- **Foundation** — DNYX Symbol + Horizon 워드마크
- **Tokens** — 색 · 타입 스케일(11단계) · 웨이트 · 라인하이트 · 트래킹 · Tone(semantic). `tokens.css`의 `:root` 단일 출처 (Tailwind 없음).
- **Primitives** — 자립 렌더 ~50개 (범용 블록 + Chrome Parts + AI Parts)
- **Composite** — 1단계 조합 ~12개 (Chrome 영역 + AI 영역)
- **Shell** — 최종 조립 2개 (DashboardLayout, DenyxAiWidget)
- **Page** — 전체 페이지 합성 템플릿 (PresetA~D)

## 5 핵심 규칙

이 5가지만 지키면 디자인 시스템의 80%를 따른 것입니다. 자세한 근거는 [patterns.md](/design-system/patterns.md).

1. **인라인 색상 리터럴 금지** — `#E53935` 같은 hex 를 컴포넌트에 직접 쓰지 마세요. `tokens.css` 의 `:root` 변수나 `TONE_*` 토큰 참조. 새 톤은 토큰에 먼저 추가.
2. **위젯 카드는 `<AiCard>` 로 감싼다** — 흰 배경 + 보더 + 8px 라운드 + 12px 패딩 + 진입 애니메이션은 `AiCard` 가 책임. div 를 새로 짜면 톤이 어긋남.
3. **AI 위젯이 열려도 페이지 헤더는 그대로** — 사이드바는 240→40px 로 축소되지만, 헤더의 title / AI / Docs / 고객지원 / bell / avatar 는 절대 사라지지 않음.
4. **차트 텍스트는 단위까지** — SVG 안의 `<text fontSize>` 는 반드시 `"9px"` 처럼 단위를 붙임. 단위 없는 숫자는 viewBox 좌표로 계산되어 화면마다 크기가 달라짐.
5. **DataTable density 기본 `default`** — header 12px / row 13px. compact (10/11) 는 100+ 행 raw log 한정. 셀 내 inline 폰트는 `text-md`(주) / `text-base`(부) 만 사용.

이 규칙(및 그 외 불변식)이 깨지지 않게 지키는 무결성 게이트는 **[Guardrails](/?path=/docs/tokens-guardrails--docs)** 참조 — 🟢 자동 / 🟡 정책 / 🔵 계획.

## 빠른 시작

### 신규 페이지 작성

```tsx
import {
  DashboardLayout, PageHeaderAiInline,
  DenyxAiWidget, useDenyxAi,
  type AiPromptSuggestion,
} from "@denyx/design-system";

const PAGE_AI_SUGGESTIONS: AiPromptSuggestion[] = [
  { label: "DB 시간대별 사용량", query: "Oracle DB 24시간 사용량 분석" },
  { label: "병목 조회 분석",     query: "느린 쿼리 TOP 5 분석" },
];

export default function MyPage() {
  const { aiActive, setAiActive } = useDenyxAi();
  return (
    <DashboardLayout
      activeProduct="db"
      widgetOpen={aiActive}
      header={<PageHeaderAiInline
        title="DB / oracle_dnx"
        promptSuggestions={PAGE_AI_SUGGESTIONS}
        onPromptSubmit={(q) => { setAiActive(true); /* 핸들 */ }}
      />}
      main={<MyContent />}
    >
      <DenyxAiWidget open={aiActive} onClose={() => setAiActive(false)} />
    </DashboardLayout>
  );
}
```

### 신규 AI 응답 메시지

```tsx
import {
  AiCard, AiSectionHeading, AiBulletList,
  type SemanticIntent,
} from "@denyx/design-system/widget";

function CriticalFinding({ intent, title, bullets }: {
  intent: SemanticIntent;
  title: string;
  bullets: string[];
}) {
  return (
    <AiCard delay={0}>
      <AiSectionHeading tone={intent}>{title}</AiSectionHeading>
      <AiBulletList items={bullets} />
    </AiCard>
  );
}
```

**짧음에 주목**하세요. 새 색이 필요 없고, 새 폰트도 없고, 새 카드 chrome 도 없습니다. 디자인 시스템이 잘 작동하면 이렇게 짧아집니다.
